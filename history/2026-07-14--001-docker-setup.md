# 001 Docker Setup

## Summary

Implemented a minimal static Ralphi web GUI served by nginx in Docker and exposed it under `/ralphi/` through the shared vibes-proxy route.

## Decisions

- Used plain HTML, CSS, and JavaScript with no frontend framework.
- Served the app with nginx and a `/healthz` endpoint so Docker healthchecks also appear in stdout access logs.
- Published the local development port as `8081:80` because port 8080 was already occupied on IPv6 by an existing Python server.
- Added `handle_path /ralphi/*` to `/Users/boot/Documents/vibes/vibes-proxy/Caddyfile` above the default Planed handler.

## Verification

- `docker compose config` passed for Ralphi and vibes-proxy.
- `docker compose up -d --build` started Ralphi successfully.
- Ralphi container health reached `healthy`.
- vibes-proxy was force-recreated and remained `healthy`.
- `http://localhost:8081/ralphi/`, `http://localhost:18080/ralphi/`, and `https://mac-mini-von-est.tailfb7eca.ts.net/ralphi/` returned the required page.
- CSS and JS assets returned HTTP 200 from the configured port, proxy, and HTTPS route.
- Browserless/Chrome rendered the page successfully and reported no console errors, failed requests, or 4xx/5xx responses.

## Issues

- `http://localhost:8080` was already bound by a Python SimpleHTTP server on IPv6, so the app uses configured port 8081 for direct local access.
- Native in-app browser control was unavailable in this session, and a macOS Safari screenshot attempt hung; Browserless/Chrome was used instead for visual verification.
